Добавляй наиболее важные для архитектуры проекта правила сюда для учета в будущих сессиях

Проект:
- TypeScript 5.x.
- Чистая архитектура: `domain`, `application`, `infrastructure`, `presentation`.
- Сборка через SWC и Rollup.
- Результат сборки должен быть одним browser-loadable JavaScript-файлом.
- Деплой конечного JavaScript-файла выполняется через GitHub Actions на GitHub Pages.

Код:
- Спагетти-код неприемлем: разбивай код на небольшие модули и функции с одной понятной ответственностью.
- Не смешивай доменную логику, сценарии приложения, инфраструктуру и UI-детали в одном месте.

Цель продукта:
- Делаем простого бота для браузерной игры.
- Бот должен помогать собирать ресурсы через сканирование локации на мобов и ресурсы.
- Реализуем функциональность пошагово, без преждевременного усложнения.

Текущий UI-ориентир:
- При загрузке скрипта на страницу показывать круглую кнопку с иконкой кирки.
- Стиль интерфейса: темная тема, лаконично, аккуратно, с ощущением игрового инструмента.
- По клику на кнопку открывать небольшое окно интерфейса бота рядом с кнопкой.
- В окне пока нужен только блок логов.
- Формат лога: `время: сообщение`.
- Окно должно закрываться крестиком и быть перетаскиваемым.
